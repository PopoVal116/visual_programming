# Visual Programming — Android Client

Android-клиент системы визуального программирования. Приложение позволяет создавать простые визуальные конструкции и отправлять их на сервер для обработки и визуализации результатов.

## Возможности

- Визуальное и функциональное программирование через отдельные модули
- Обмен данными с сервером в реальном времени
- Работа с геолокацией устройства
- Воспроизведение медиафайлов
- Тестирование сетевого взаимодействия

## Стек технологий

- **Язык**: Kotlin
- **Минимальная версия Android**: SDK 21
- **Сборка**: Gradle (Kotlin DSL)
- **Сетевое взаимодействие**: ZeroMQ (jeromq)
- **Сериализация**: Gson
- **Геолокация**: Google Play Services Location
- **Архитектура**: Jetpack компоненты (ViewModel, LiveData / StateFlow — готово к миграции на Compose)

## Основные Activity

| Activity                | Назначение                                                                 |
|-------------------------|----------------------------------------------------------------------------|
| **MainActivity**        | Главный экран приложения. Служит точкой входа и обеспечивает навигацию между модулями. |
| **SocketActivity**      | Основная Activity для сетевого взаимодействия. Реализует подключение к серверу, отправку и приём сообщений через ZeroMQ. |
| **CalculatorActivity**  | Модуль калькулятора. Позволяет выполнять математические вычисления в рамках визуального программирования. |
| **LocationActivity**    | Модуль геолокации. Получает текущие координаты устройства и может передавать их на сервер. |
| **MediaPlayerActivity** | Медиаплеер. Поддерживает воспроизведение аудио и видео файлов как одну из демонстрационных возможностей системы. |


## Скриншоты

MainActivity — главный экран

SocketActivity — взаимодействие с сервером

CalculatorActivity — калькулятор

LocationActivity — геолокация

MediaPlayerActivity — медиаплеер

Связь с сервером
Приложение работает в связке с репозиторием visual_server.
Обмен данными осуществляется по протоколу ZeroMQ. Подробное описание протокола и форматов сообщений находится в документации сервера.
